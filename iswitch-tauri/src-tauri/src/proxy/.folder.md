# Proxy 模块

## 功能定位

[POS]: HTTP 代理服务器核心模块，基于 axum 实现本地代理，转发请求到 Provider API

## 架构设计

```
              ┌─────────────────────────────────────┐
              │          Proxy Server (:18099)      │
              ├─────────────────────────────────────┤
              │                                     │
Client ──────▶│  router.rs ───▶ handler.rs ────────▶│───▶ Provider API
              │       │              │              │
              │       │         health.rs           │
              │       └───▶ middleware.rs           │
              │                  │                  │
              │                  ▼                  │
              │            (CORS, Trace)            │
              └─────────────────────────────────────┘
```

## 文件清单

| 文件            | [POS] 定位                                    | 迁移自                           |

## 依赖关系

- [INPUT]: ../services/provider_service.rs - 获取 Provider 列表
- [INPUT]: ../services/log_service.rs - 记录请求日志
- [INPUT]: ../models/provider.rs - Provider 数据结构
- [OUTPUT]: 提供 HTTP API 给 Claude Code / Codex

## 关键功能

1. **多供应商路由**: 根据优先级选择可用 Provider
2. **自动降级**: Provider 失败时自动切换下一个
3. **智能健康检查**: 基于连续失败次数自动降级不健康供应商
4. **惰性恢复**: 降级供应商在 recovery_timeout 后自动尝试恢复
5. **保底策略**: 当所有供应商都降级时，忽略降级状态强制尝试
6. **流式转发**: SSE 响应实时转发，不缓存全量
7. **Token 解析**: 从响应中提取 Usage 信息

## 规范遵循

- [PROTOCOL]: FractalFlow v1.0
- 使用 `reqwest` 进行 HTTP 请求
- 使用 `axum::body::Body::from_stream` 转发流式响应
