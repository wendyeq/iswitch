# Commands 模块

## 功能定位

[POS]: Tauri Commands 层 - 前后端 IPC 桥接，定义所有可供前端调用的 Rust 函数

## 设计原则

1. **薄层设计**: Commands 只做参数解包和 Service 调用，不包含业务逻辑
2. **类型安全**: 所有参数和返回值必须是强类型，避免 `serde_json::Value`
3. **错误传递**: 统一返回 `AppResult<T>`，自动序列化给前端

## 文件清单

| 文件          | [POS] 定位                 | 迁移自         |
| :------------ | :------------------------- | :------------- |
| `mod.rs`      | 模块导出                   | -              |
| `provider.rs` | Provider 相关 Commands     | Wails bindings |
| `claude.rs`   | Claude 设置 Commands       | Wails bindings |
| `codex.rs`    | Codex 设置 Commands        | Wails bindings |
| `mcp.rs`      | MCP Server 管理 Commands   | Wails bindings |
| `skill.rs`    | Skill 管理 Commands        | Wails bindings |
| `log.rs`      | 日志查询 Commands          | Wails bindings |
| `hud.rs`      | Mini HUD 窗口管理 Commands | 新增           |
| `settings.rs` | 应用设置 Commands          | Wails bindings |

## 依赖关系

- [INPUT]: ../services/ - 调用 Service 层执行业务逻辑
- [INPUT]: ../models/ - 使用数据模型作为参数和返回值
- [OUTPUT]: 前端 `@tauri-apps/api/core` invoke 调用

## 使用示例 (前端)

```typescript
import { invoke } from '@tauri-apps/api/core';

const providers = await invoke<Provider[]>('load_providers', { kind: 'claude' });
```
