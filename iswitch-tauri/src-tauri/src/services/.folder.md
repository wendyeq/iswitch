# Services 模块

## 功能定位

[POS]: 业务逻辑服务层，封装所有核心功能模块的实现

## 设计原则

1. **单一职责**: 每个 Service 只负责一个功能域
2. **可测试性**: Service 方法应便于单元测试，外部依赖通过参数或 trait 注入
3. **状态共享**: 使用 `Arc<RwLock<T>>` 在多线程环境下共享状态
