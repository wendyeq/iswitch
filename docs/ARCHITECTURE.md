## 2. 技术栈

| 层级     | 技术               | 用途              |
| :------- | :----------------- | :---------------- |
| **前端** | Vue 3 + TypeScript | UI 框架与类型安全 |
|          | TailwindCSS        | 样式系统          |
|          | Vue Router 4       | 路由管理          |
|          | Vue I18n           | 国际化            |
|          | Vite               | 构建工具          |
| **后端** | Rust               | 系统编程语言      |
|          | Tauri v2           | 桌面应用框架      |
|          | axum               | HTTP 代理服务器   |
|          | SQLite             | 本地数据存储      |
|          | reqwest            | HTTP 客户端       |
|          | tracing            | 日志系统          |

## 5. 核心模块说明

### 5.1 Proxy 模块 (代理服务)

负责本地HTTP代理，实现请求转发与智能路由。

| 文件         | 职责                                   |
| :----------- | :------------------------------------- |
| `server.rs`  | axum服务器启动/关闭控制                |
| `router.rs`  | API路由定义 (/v1/messages, /responses) |
| `handler.rs` | 核心请求处理与Provider选择逻辑         |
| `health.rs`  | Provider健康状态追踪与智能降级         |
| `monitor.rs` | 流式响应监控、Token解析、日志记录      |

**关键特性**:

- ✅ 多供应商优先级路由
- ✅ 连续失败自动降级 (默认: 3次)
- ✅ 惰性恢复机制 (默认: 5分钟)
- ✅ 保底策略 (全部降级时强制重试)
- ✅ SSE流式响应实时转发

### 5.2 Services 模块 (业务逻辑)

封装所有核心业务功能的实现。

| Service            | 职责                 |
| :----------------- | :------------------- |
| `provider_service` | Provider配置CRUD操作 |
| `claude_settings`  | Claude Code配置读写  |
| `codex_settings`   | Codex配置读写        |
| `mcp_service`      | MCP Server管理       |
| `skill_service`    | Skill下载安装        |
| `log_service`      | 请求日志与统计       |
| `app_settings`     | 应用设置持久化       |
| `import_service`   | 旧版配置导入         |
| `pricing_service`  | 模型定价与成本计算   |

### 5.3 Commands 模块 (前后端桥接)

Tauri Commands层，暴露Rust功能给前端调用。

| Command       | 对应Service                                   |
| :------------ | :-------------------------------------------- |
| `provider::*` | provider_service                              |
| `settings::*` | claude_settings, codex_settings, app_settings |
| `log::*`      | log_service                                   |
| `mcp::*`      | mcp_service                                   |
| `skill::*`    | skill_service                                 |
